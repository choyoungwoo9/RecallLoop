# 학습 서비스 사용자 스토리 (구체화)

## 1. 학습 기록

### 1.1 기록 작성

- 사용자는 배운 내용을 잊지 않기 위해 학습 내용을 기록하고 싶다.
- 사용자는 기록을 작성할 때 제목과 내용을 입력할 수 있다.
- 사용자는 강의, 책, 블로그 등에서 배운 내용을 복사해서 붙여넣어 저장할 수 있다.

### 1.2 기록 조회

- 사용자는 저장한 학습 기록 목록을 확인할 수 있다.
- 사용자는 특정 기록을 선택하여 내용을 다시 볼 수 있다.

### 1.3 기록 관리

- 사용자는 기록을 삭제할 수 있다.

---

## 2. 문제 관리

### 2.1 문제 생성 설정

- 사용자는 학습 기록마다 문제 생성 설정을 만들 수 있다.
- 사용자는 "어떤 문제를 만들고 싶은지"에 대한 설명을 입력할 수 있다.
- 사용자는 이 설명을 한 줄 정도의 간단한 문장으로 입력할 수 있다.
- 사용자는 하나의 학습 기록에 대해 여러 개의 문제 생성 설정을 만들 수 있다.
- 사용자는 몇 개의 문제를 만들 것인지 입력할 수 있다.

예시

- "핵심 개념을 설명하는 문제를 만들어줘"
- "이 글에서 중요한 키워드를 묻는 문제를 만들어줘"
- "내용을 보고 개념을 떠올릴 수 있는 문제를 만들어줘"

### 2.2 문제 생성

- 사용자는 설정한 문제 생성 방식에 따라 학습 기록 기반 문제를 생성할 수 있다.
- 사용자는 설정한 문제 수만큼 문제가 생성되기를 원한다.

### 2.3 문제 관리

- 사용자는 생성된 문제 목록을 확인할 수 있다.
- 사용자는 문제가 적절하지 않다고 판단하면 문제를 삭제할 수 있다.

---

## 3. 문제 진행 방식

### 3.1 순차 풀이

- 사용자는 문제를 별도로 고르지 않고 순서대로 풀고 싶다.
- 사용자는 문제를 queue 방식으로 순차적으로 받고 싶다.
- 사용자는 문제를 제출하면 다음 문제가 이어지기를 원한다.
- 사용자는 어떤 문제를 풀지 매번 판단하지 않고 정해진 순서대로 학습하고 싶다.

### 3.2 반복 순환

- 사용자는 문제 전체를 한 바퀴 돌고 나면 다시 처음부터 반복해서 풀고 싶다.
- 사용자는 문제를 뺑뺑이 방식으로 계속 순환하며 복습하고 싶다.

### 3.3 진행 상태 확인

- 사용자는 현재 queue에서 몇 번째 문제를 풀고 있는지 알고 싶다.
- 사용자는 전체 문제 중 어디까지 진행했는지 확인하고 싶다.
- 사용자는 한 바퀴를 얼마나 진행했는지 확인하고 싶다.

---

## 4. 문제 풀이

### 4.1 풀이 인터페이스

- 사용자는 문제를 보고 답을 직접 입력할 수 있다.
- 사용자는 답을 제출할 수 있다.
- 사용자는 문제를 풀 때 충분히 생각할 수 있기를 원한다.

### 4.2 정답 처리

- 사용자는 정답 여부를 자동으로 채점받지 않는다.
- 사용자는 문제 제출 후 바로 다음 문제로 넘어간다.

### 4.3 기록 기반 확인

- 사용자는 특정 학습 기록에 포함된 문제를 모두 풀었을 때 해당 학습 기록을 다시 확인할 수 있는 링크를 제공받는다.
- 사용자는 링크를 통해 학습 기록을 다시 읽으며 자신의 답을 스스로 검증할 수 있다.
- 사용자는 학습 기록을 바로 확인하거나 다음 문제로 넘어갈지 선택할 수 있다.

### 4.4 풀이 기록

- 사용자는 문제를 제출한 기록이 남기를 원한다.
- 사용자는 문제 풀이 시간을 기록하고 싶다.

---

## 5. 복습 관리

### 5.1 반복 복습

- 사용자는 문제를 한 번만 풀고 끝내지 않고 반복해서 복습하고 싶다.
- 사용자는 queue를 계속 순환하면서 자연스럽게 복습하고 싶다.

### 5.2 문제 순환

- 사용자는 모든 문제가 queue 안에서 계속 반복되기를 원한다.
- 사용자는 특정 문제만 따로 관리하지 않고 전체 문제를 순환하며 학습하고 싶다.
